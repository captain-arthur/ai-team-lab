# Review Notes(계획 검토): Ginkgo 조사/실험 타당성

**Date:** 2026-03-18

## 강점
- 평가축이 CAT 최소 조건 4개에 직접 연결되어 “조사”가 아니라 “판단에 필요한 실험 포인트”로 좁혀져 있다.
- 최소 실험 후보가 프레임워크 제약/경계(단언, 결과 저장, 시나리오 주입)를 각각 직접 찌르도록 구성됐다.
- k6/CL2와의 역할 경계를 후보 C로 명확히 확인하려고 한다.

## 아직 남은 모호한 부분(다음 단계 구현 전에 확인)
- Ginkgo 결과 파일을 CAT 표준 스키마로 정규화하는 데 필요한 최소 변환 로직이 얼마나 단순한지(어떤 훅/리포터를 쓸 수 있는지) 아직 확인 전이다.
- “오케스트레이션 기반까지”는 후보 C 이후 판단이 필요하므로, 어떤 수준에서 플랫폼 기반이라고 부를지 기준을 다음 단계에서 고정해야 한다.

## 구현 가능한 수준인지(이 단계 기준)
- **YES(다음 단계로 넘어갈 수 있음)**: 이 문서는 구현 없이도 “무엇을 실험할지”가 구체적이다.
- 다만 실제로 실행/구현하려면 Ginkgo 리포터/기본 출력 구조와 커스텀 파일 출력 hook을 확인해야 한다.

## 구현 전 추가 확인이 필요한 것
- Ginkgo가 테스트 종료/패스 여부를 외부에 전달하는 방식(종료 코드, hooks, 리포트 생성 경로)
- JUnit XML/JSON 출력이 CAT 소비에 바로 쓸 수 있는지(또는 어댑터가 필요한지)
