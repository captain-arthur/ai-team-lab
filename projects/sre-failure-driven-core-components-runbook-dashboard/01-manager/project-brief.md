# 프로젝트 브리프: 장애 주도 운영 대시보드 + Runbook 연계 설계 완성

## 이번 프로젝트의 목적
운영 모니터링에서 “관측(visibility)”은 이미 충분한 경우가 많다.  
그런데 운영자는 여전히 다음 질문에 빠르게 답하지 못한다.

1. 지금 안전한가?
2. 내일도 안전한가?
3. 지금 무엇을 해야 하는가?

이 프로젝트는 위 질문을 단순 metric 나열이 아니라, **장애(실패) 중심 판정 → 경고/위험 → 즉시 실행 가능한 runbook**으로 연결되는 구조로 확정한다.

## 왜 runbook 연계가 중요한가
- “이상 있음”을 보여주는 것만으로는 행동이 늦어진다.
- 운영자는 상황 판단과 검색을 동시에 해야 해서, 5~10분 사이의 골든 타임을 잃는다.
- 따라서 패널의 상태가 곧 다음 행동을 만들도록, **대시보드와 runbook의 접점을 고정**해야 한다.

## 최종 산출물이 무엇이어야 하는가
1. “장애 주도 대시보드”와 “대시보드 주도 액션”의 정의를 실제 패널 구조/판정 로직으로 번역한 문서
2. `istio-ingressgateway / nodelocaldns / coredns`에 대해 Grafana로 바로 그릴 수 있는 최종 row/panel 청사진
3. 각 컴포넌트별 최소 runbook starter(대시보드 상태에 직접 연결)
4. 운영자가 실제로 5~10분 내 분류/행동 가능한지에 대한 실무 타당성 검토

## 성공 기준
- 철학과 방법론이 정리되고, 그 결과가 **바로 대시보드 설계로 내려와야 한다**
- runbook 연결 구조가 패널 수준에서 명확히 정의되어야 한다
- 문서만 보고도 실제로 Grafana row/panel을 그릴 수 있을 정도로 구체적이어야 한다
- 기술적/운영적 타당성이 함께 검증되어야 한다
