# k6를 CAT 시스템에 통합하는 설계

**Date:** 2026-03-18  
**Program:** CAT

## 목적
- k6를 CAT의 “테스트 도구”로 편입하기 위한 **실행 가능한 통합 설계**를 정의한다.

## 해결하려는 질문
👉 “k6는 CAT 시스템 안에서 어떤 위치에 들어가며, 어떤 방식으로 실행/판정/저장되어야 하는가?”

## 고정 성공 기준(CAT 최소 요구 4요소)
설계가 아래 4가지를 **모호함 없이** 만족해야 한다.
1. **시나리오 주입**: CAT가 시나리오(타깃/부하/옵션)를 k6 실행에 주입 가능
2. **SLI 측정**: 선택된 SLI(예: p95 latency, 실패율)를 기계적으로 추출 가능
3. **SLO 평가**: 결과가 “수치”가 아니라 **PASS/FAIL 단언**으로 남음(근거 포함)
4. **결과 저장**: 최종 결과가 **파일로 저장**됨(JSON/YAML/JUnit 등)

## 범위
- **포함**: k6 통합 역할/경계, 결과 파일 형식, PASS/FAIL 설계, Prometheus의 위치(보조/승격), CL2와 관계
- **제외**: CAT 전체 시스템 구현, UI/시각화 구현, 특정 클러스터(ingress) 적용 튜닝
