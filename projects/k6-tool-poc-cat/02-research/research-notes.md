# Research Notes: k6 (CAT 관점의 도구 이해)

**Date:** 2026-03-18

## 1) k6는 무엇인가 (누가/왜)
- **만든 주체**: Load Impact에서 시작 → 현재 Grafana Labs에서 유지/발전.
- **만든 이유**: 코드(스크립트)로 **부하/시나리오를 정의**하고, 실행 결과로 **성능/신뢰성 지표를 측정**해 자동 판정까지 가능하게 하려는 목적.

## 2) 핵심 구성요소
- **VU (Virtual User)**: 스크립트를 실행하는 동시 실행자(작은 워커).
- **Iteration**: VU가 “한 번” 시나리오 코드를 수행한 단위(사용자 1회 행동).
- **Scenario**: 어떤 실행자(executor)로, 얼마만큼의 동시성/도착률로, 얼마 동안 어떤 함수를 실행할지 정의한 **부하 모델**.

## 3) 실행 흐름(최소)
- **init(모듈 스코프)**: 옵션/시나리오/threshold 정의, 데이터 로딩
- **VU 코드(default 또는 scenario exec)**: 각 VU가 반복 수행(= iteration 누적)
- (선택) **setup/teardown**: 실행 전/후 1회 로직

## 4) k6가 해결하는 문제 / 못하는 것
- **해결**: “정의된 클라이언트 부하”를 생성하고, 그 결과를 **SLI(예: p95 latency, 실패율)**로 요약 + threshold로 **PASS/FAIL**까지 자동화.
- **못함(기본 설계 밖)**: 실제 브라우저 기반 UI 테스트, 내부 클러스터 상태 자체의 검증(예: 컨트롤플레인 SLI) 같은 **내부 관측 중심 합격 판정**.

## 5) CAT 관점으로 번역(역할/경계)
- **k6의 역할**: “클러스터 바깥에서” 클라이언트 트래픽을 만들고, 그 관측값으로 **외부 SLI**를 만들며, **SLO(threshold)로 PASS/FAIL**을 내는 도구.
- **k6가 잘하는 것**: (a) 시나리오(부하 모델) 정의, (b) 요청 단위 SLI 집계(p95/실패율/요청률), (c) 자동 판정.
- **k6가 못하는 것**: 내부 구성요소의 건강/자원/제어면 지표 같은 **내부 SLI 기반 합격 판정**(측정/판정의 주체가 아님).
