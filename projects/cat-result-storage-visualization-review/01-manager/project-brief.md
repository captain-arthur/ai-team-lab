# CAT 결과 저장/시각화 구조 재검토(파일 vs Evidence/InfluxDB)

**Date:** 2026-03-19

## 이번 검토의 목적
- CAT 결과를 **파일 기반만** 저장할지, **InfluxDB 시계열 저장소 중심**으로 갈지, 또는 **혼합**할지
  - “단순성”
  - “도구 수용성(k6/CL2/Ginkgo)”
  - “Evidence 시각화 가능성”
  - “운영 부담”
  관점에서 가장 실용적인 결정을 내린다.

## 무엇을 비교/판단할 것인가
1. Evidence로 대시보드/리포트를 만들 때 필요한 데이터 흐름
2. CAT의 최종 판정 결과(`cat-result.json`)와 시계열 관측 데이터의 역할 분리 여부
3. k6/CL2/Ginkgo 각각에 대해 adapter/수집 복잡도가 얼마나 늘어나는지

## 성공 기준
- 3가지 옵션(A/B/C)을 CAT 관점에서 비교 가능
- Evidence를 “CAT 결과 시각화”에 실제로 붙일 수 있는지 판단 가능
- 최종 추천안이 하나로 정리됨(또는 단계적 전략이 명확함)
