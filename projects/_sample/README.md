# Sample Project: Local CLI Task Tracker

This folder is an **example project** showing how the AI team workflow is used end-to-end. It is not a real deliverable; it demonstrates structure and handoffs.

---

## Original request (example)

> "I want a simple way to track personal tasks from the terminal. Prefer a single binary or script, minimal setup, and plain-text storage so I can sync with Dropbox. I don't need a UI or a server — CLI only."

---

## Goal (summary)

Choose and document a **local, CLI-only task tracker** with plain-text storage and minimal setup, and produce a short report plus usage notes so the user can adopt it.

---

## How to read this sample

1. **01-manager/** — Brief, work breakdown, and handoffs for each role.
2. **02-research/** — Comparison of a few CLI task tools and a recommendation.
3. **03-architecture/** — "Architecture" here is minimal (single user, local only) but shows the template.
4. **04-engineering/** — Example implementation plan and a tiny script or config.
5. **05-review/** — Example review checklist and issues.
6. **06-documentation/** — Example final report and user-facing README.

Use this as a reference when creating your own projects under `projects/<your-project>/`.
