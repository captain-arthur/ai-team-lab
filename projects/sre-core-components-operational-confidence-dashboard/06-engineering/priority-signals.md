# 우선 신호 인벤토리(컴포넌트별 TOP 5~10)

여기서의 “signal”은 metric 나열이 아니라, 각 컴포넌트의 운영 판단에 직접 쓰이는 evidence 단위다.
각 signal마다 “지금 안전성”, “내일 안전성 선행”, “행동 유도” 중 어디에 쓰이는지까지 명시한다.

## 1) istio-ingressgateway 신호 TOP
1. 요청 성공률(사용자 관측 성공 비율)  
   - 지금 안전: fail/partial fail을 가장 먼저 드러낸다.  
   - 행동 유도: 성공률이 떨어질 때 “게이트웨이 헬스 vs upstream” 분기 기준이 된다.
2. 요청 타임아웃 비율  
   - 지금 안전: “응답 품질 붕괴”의 핵심 지표다.  
   - 내일 안전: 타임아웃이 늘어나는 추세는 용량/서킷차단 문제로 이어질 수 있다.
3. p95/p99 요청 지연(가능하면 오류율과 함께)  
   - 지금 안전: latency 상승은 사용자 영향으로 바로 전이된다.  
   - 행동 유도: 지연만 오르고 오류가 낮으면 upstream 병목/큐잉을 먼저 본다.
4. 업스트림 연결 실패/라우팅 실패 계열(게이트웨이에서 보이는 upstream 실패)  
   - 지금 안전: 내부 서비스로의 전달 품질을 보여준다.  
   - 행동 유도: upstream 문제면 ingressgateway 자체 조정보다 “타깃 서비스/엔드포인트 상태”가 우선이다.
5. 부하 완화/서킷차단/드롭의 존재 여부(로드가 걸리면 반드시 튀는 “전이” 신호)  
   - 내일 안전: 현재는 괜찮아 보여도, 완화 신호가 반복되면 내일 위험으로 승격한다.  
   - 행동 유도: 완화/차단이 관측되면 우선순위를 “용량/정책(타임아웃·라우팅)”으로 옮긴다.
6. 게이트웨이 준비성/헬스 실패(ready 불안정, 재시작)  
   - 지금 안전: 준비성 불안정은 즉시 사용자 영향으로 이어질 수 있다.  
   - 행동 유도: 준비성 실패가 크면 로그/리소스(특히 프록시 CPU/메모리)부터 본다.

## 2) nodelocaldns 신호 TOP
1. DNS 질의 성공률(애플리케이션이 체감하는 이름 해석 성공)  
   - 지금 안전: DNS가 깨지면 서비스 통신 전체가 흔들린다.  
   - 행동 유도: 성공률이 떨어지면 “로컬 문제 vs upstream(coredns) 문제”부터 분기한다.
2. DNS 타임아웃 비율  
   - 지금 안전: 타임아웃은 재시도 폭발로 전이를 만든다.  
   - 내일 안전: 시간에 따라 타임아웃이 누적/확산되면 risk-increasing으로 승격한다.
3. 응답 지연 p95(노드 로컬 처리 품질)  
   - 지금 안전: 동일 서비스라도 노드 편차가 있으면 “영향 범위”를 즉시 추정할 수 있다.  
   - 행동 유도: 지연이 특정 노드군에 집중되면 그 노드군/네트워크 경로를 먼저 본다.
4. upstream(coredns) 포워딩 실패/에러율 계열  
   - 내일 안전: 로컬은 버티지만 upstream이 흔들리면 점진적으로 위험이 커진다.  
   - 행동 유도: upstream 실패가 주 원인이면, coredns Row로 우선 조사 방향이 바뀐다.
5. 캐시 효과 지표(캐시 hit이 줄고 miss가 증가하는 경향)  
   - 내일 안전: 캐시가 효율적으로 동작하지 않으면 질의 부하가 upstream으로 전이된다.  
   - 행동 유도: miss 증가가 지속되면 nodelocaldns 설정/리소스와 upstream 확장을 검토한다.
6. nodelocaldns 포드 재시작/리소스 압박(잦은 evict 포함)  
   - 지금 안전: 노드 로컬 DNS 자체가 불안정하면 “빠른 안정화 조치”가 필요하다.  
   - 내일 안전: 재시작 빈도가 계속되면 risk-high로 간주한다.

## 3) coredns 신호 TOP
1. DNS 에러율(실패 유형 중 운영 판단에 중요한 계열: SERVFAIL/타임아웃 등)  
   - 지금 안전: 클러스터 전반의 이름 해석 실패를 직접 반영한다.  
   - 행동 유도: 특정 에러 유형이 늘면 원인(업스트림 forward/플러그인/설정)을 분류하는 단서가 된다.
2. DNS 응답 지연 p95/p99  
   - 지금 안전: 지연이 길어지면 재시도가 늘고, 애플리케이션은 타임아웃으로 이동한다.  
   - 내일 안전: p99가 회복되지 않으면 risk-increasing으로 올린다.
3. 처리 포화 징후(큐잉/인플라이트 증가, 워커 소진)  
   - 내일 안전: “품질 악화로 가는 전이”를 가장 빨리 보여준다.  
   - 행동 유도: 포화가 확인되면 스케일/리소스 또는 트래픽(질의량) 측면을 먼저 본다.
4. 캐시 관련 지표(캐시 적중률 저하/에비션 증가)  
   - 내일 안전: 캐시 효율이 떨어지면 upstream 의존이 커져 위험이 누적된다.  
   - 행동 유도: 캐시 저하가 원인이면 구성/리소스/쿼리 패턴을 같이 점검한다.
5. 헬스 엔드포인트/준비성 상태(ready flapping 포함)  
   - 지금 안전: coredns가 불안정하면 nodelocaldns가 upstream을 기다리며 지연이 누적된다.  
   - 행동 유도: ready flapping이면 즉시 로그에서 근본 원인 분류를 시작한다.
6. 재시작/로드 실패/구성 변경과의 동시성(변경이 원인일 가능성)  
   - 지금 안전: 구성 변경 직후 에러/지연이 늘면 롤백 우선순위가 올라간다.  
   - 행동 유도: 변경 히스토리와 교차검증하는 “판정 단위”가 된다.
